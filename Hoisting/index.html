🎭 Hoisting क्या है?

JavaScript कोड चलाने से पहले ही सारे variables और functions को memory में allocate कर लेता है। यह process को "Hoisting" कहते हैं।

🧠 Memory Allocation Process

Variables
Memory में Reserved
Functions

📝 Hoisting Process Timeline

1
Compilation Phase: JavaScript engine पहले code को scan करता है
2
Memory Allocation: Variables और functions के लिए memory reserve करता है
3
Execution Phase: अब actual code execution शुरू होता है

🔄 var Hoisting का जादू

console.log(babitaji); // undefined
var babitaji;
babitaji = "Beautiful";
console.log(babitaji); // "Beautiful"
var babitaji; // Hoisted to top
console.log(babitaji); // undefined
babitaji = "Beautiful";
console.log(babitaji); // "Beautiful"
जेठालाल जी

देखिए बबिता जी, var के साथ variable का नाम तो ऊपर चला जाता है, लेकिन value वहीं रहती है जहाँ आपने assign की है। इसलिए पहले undefined मिलता है!

⚡ let & const: Temporal Dead Zone

बबिता जी

अगर let और const भी hoist होते हैं, तो फिर error क्यों आता है जब हम उन्हें ऊपर से access करते हैं?

जेठालाल जी

बहुत बढ़िया सवाल! let और const को JavaScript hoist तो कर देती है, लेकिन उन्हें Temporal Dead Zone में रखती है। मतलब जब तक आप manually उन्हें initialize नहीं करते, तब तक वो access नहीं हो सकते!

console.log(jethalal); // ReferenceError!
let jethalal = "Tapu ke Papa";
console.log(jethalal); // "Tapu ke Papa"
//let example
console.log(SOCIETY); // ReferenceError!
const SOCIETY = "Gokuldham";
console.log(SOCIETY); // "Gokuldham"

🎯 Function Hoisting

जेठालाल जी

Function declarations पूरी तरह hoist होती हैं — इसलिए उन्हें पहले call कर सकते हैं। लेकिन function expressions बिल्कुल variables की तरह behave करती हैं!

sayHello(); // "Hello from Gokuldham!"
function sayHello() {
return "Hello from Gokuldham!";
}
sayBye(); // TypeError!
var sayBye = function() {
return "Bye from Gokuldham!";
};

❌ Variable Hoisting
Arrow Function
