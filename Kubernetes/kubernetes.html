💻 Kubernetes YAML ke Samples

# 🚀 Bhide ki Website ka Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
name: bhide-achaar-website
labels:
app: achaar-papad
spec:
replicas: 3 # 3 copies hamesha ready
selector:
matchLabels:
app: achaar-papad
template:
metadata:
labels:
app: achaar-papad
spec:
containers:
- name: website
image: bhide/achaar-website:v1.0
ports:
- containerPort: 80
resources:
requests:
cpu: 100m
memory: 128Mi
limits:
cpu: 500m
memory: 512Mi
Bhide ki website deploy karne ke liye click karo...
# 🌐 Load Balancer Service
apiVersion: v1
kind: Service
metadata:
name: bhide-achaar-service
spec:
type: LoadBalancer
selector:
app: achaar-papad
ports:
- port: 80
targetPort: 80
protocol: TCP
# Payment ke liye Internal Service
---
apiVersion: v1
kind: Service
metadata:
name: payment-service
spec:
type: ClusterIP # Internal service
selector:
app: payment-app
ports:
- port: 8080
targetPort: 8080

# 📈 Auto-Scaling ka Jadoo
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
name: bhide-achaar-hpa
spec:
scaleTargetRef:
apiVersion: apps/v1
kind: Deployment
name: bhide-achaar-website
minReplicas: 2 # Hamesha 2 pods toh banta hai
maxReplicas: 20 # Festival sale ke liye 20 tak
metrics:
- type: Resource
resource:
name: cpu
target:
type: Utilization
averageUtilization: 70 # 70% CPU pe scale up
- type: Resource
resource:
name: memory
target:
type: Utilization
averageUtilization: 80 # 80% memory pe scale up
# Behavior (optional)
behavior:
scaleUp:
stabilizationWindowSeconds: 60
policies:
- type: Percent
value: 100
periodSeconds: 60
Auto-scaling chalu karne ke liye click karo...
