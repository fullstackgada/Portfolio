👨‍🎓 Students Table Example

Key Points:
🗝️ Primary Key: StudentID - Har student ko uniquely identify karta hai
🔄 Alternate Key: AadharNo - Yeh bhi unique hai par Primary nahi bana
🎯 Candidate Keys: StudentID, Email, AadharNo - Sabhi unique identifiers hain

Key Points:
🧩 Composite Key: (StudentID + SubjectCode) - Dono milkar unique pehchaan
🔗 Foreign Key: StudentID - Students table ke Primary Key se linked
🦸 Super Key: (StudentID + SubjectCode + Grade) - Extra column ke saath bhi unique

🔗 Table Relationships Visualization

/* Students Table */
CREATE TABLE Students (
StudentID INT PRIMARY KEY,
Name VARCHAR(50),
Email VARCHAR(100) UNIQUE,
AadharNo VARCHAR(12) UNIQUE
);
/* Enrollment Table */
CREATE TABLE Enrollment (
StudentID INT,
SubjectCode VARCHAR(10),
Grade VARCHAR(5),
PRIMARY KEY (StudentID, SubjectCode),
FOREIGN KEY (StudentID) REFERENCES Students(StudentID)
);

🔍 SQL Keys ki Takkar

Key Type Definition Properties Jethalal ki Analogy
🗝️ Primary Key Har record ko uniquely identify karti hai Not NULL, Unique, Ek table mein ek hi Secretary ki seat - Sirf ek hi banda ho sakta hai
🔗 Foreign Key Ek table ke Primary Key se link karti hai NULL ho sakti hai, Referential integrity maintain karti hai Madhvi se tera rishta hai
🎯 Candidate Key Koi bhi column jo records ko uniquely identify kar sake Ek table mein multiple, Unique, Minimal Secretary banne ke eligible candidates
🧩 Composite Key Do ya zyada columns ka combination Sath mein unique, Individually unique nahi bhi ho sakte Teri shaadi - Tu + Madhvi bhabhi dono chahiye
🦸 Super Key Koi bhi combination jo uniquely identify kare Extra unnecessary columns ho sakte hain Sakharam ka papa + Secretary = Bhide
🔄 Alternate Key Candidate Key jo Primary nahi bani Unique par Primary nahi chuni gayi Babita ji bhi candidate thi - par Alternate ban gayi
