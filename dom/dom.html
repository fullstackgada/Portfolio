üöÄ DOM Performance & Best Practices

Practice ‚úÖ Recommended ‚ùå Avoid Why?
Element Selection Cache elements: const btn = document.getElementById('btn') Repeated selection: document.getElementById('btn') DOM queries are expensive, cache reduces repeated lookups
Batch DOM Updates Use DocumentFragment or batch updates Individual DOM modifications in loops Each DOM change triggers reflow/repaint
Event Delegation Add event listener to parent, check event.target Add individual listeners to each child element Reduces memory usage and handles dynamic elements
CSS Classes vs Inline Styles Use classList.add/remove for styling Multiple individual style property changes CSS classes are more performant and maintainable
Content Updates Use textContent for plain text Using innerHTML for simple text changes innerHTML parsing is slower and can be security risk
Memory Management Remove event listeners when elements are removed Leaving event listeners attached to removed elements Prevents memory leaks and improves performance

Method Purpose Returns Use Case
getElementById() Select element by ID Single element or null When you know the exact ID
querySelector() Select first matching CSS selector Single element or null Complex selectors, first match needed
querySelectorAll() Select all matching CSS selectors NodeList (array-like) Multiple elements, bulk operations
innerHTML Get/set HTML content HTML string When you need to add HTML tags

createElement() Create new DOM element New element node Dynamic content creation
